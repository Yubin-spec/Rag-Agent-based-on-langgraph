# 项目经历简历版

## 项目名称
海关知识库智能问答系统

## 项目简介
基于 LangGraph、FastAPI、RAG 与 Text2SQL 构建海关业务智能问答平台，支持政策解读、流程咨询、结构化数据查询、多轮对话、人工介入、会话长期记忆、效果监控与用户反馈闭环。

## 个人职责
- 负责整体技术架构设计与核心功能实现，搭建 `FastAPI + LangGraph + RAG + Text2SQL + PostgreSQL + Redis + Milvus` 的智能问答系统。
- 负责 DeepSeek 大模型统一接入，设计并实现多 endpoint 负载均衡、异常切换、熔断恢复与路由状态监控机制。
- 负责知识问答链路设计，构建 `FAQ 精准匹配 + Text2SQL + RAG` 的多策略问答体系，提升不同问题类型下的准确率与响应效率。
- 负责海关场景 Prompt 工程建设，设计并实现 40+ 场景化 Prompt 模板，支持按业务问题自动匹配场景约束。
- 负责 RAG 幻觉控制机制建设，通过“严格基于检索回答 + 证据引用 + grounding 低相关重生成 + 兜底拒答”降低错误回答风险。
- 负责文档解析与入库质量保障，设计多格式（Markdown / PDF / Word）文档校验机制，涵盖表格结构校验、文档结构化信息校验、乱码检测与结构化 chunk_id 编号规范。
- 负责数据库连接韧性设计，构建统一连接池管理层，实现连接复用、自动重试（指数退避）、熔断保护与优雅降级，保障高并发与数据库故障下的系统可用性。
- 负责会话状态与长期记忆设计，将对话历史、运行态、待确认 SQL、人工中断状态持久化到 PostgreSQL，支持重启恢复。
- 负责问答效果监控与反馈分析闭环，建设 QA 观测、RAG trace、用户点赞点踩、差评案例统计与分析接口。

## 关键技术方案
- 采用 LangGraph 构建多智能体编排流程，按 `supervisor -> chat / knowledge / human` 路由不同任务，支持人工介入与 interrupt 恢复。
- RAG 检索采用 `BM25 + 向量检索 + BGE Reranker` 混合策略，通过 `3:7` 融合、重排、相关性评估与最多 3 次重检提升召回质量。
- 大模型生成前注入检索证据与场景模板，生成后使用答案-文档相关度校验；若相关度低或缺少证据编号，则自动重生成，最多 3 次。
- 对话管理采用“短期 MemorySaver + PostgreSQL 长期记忆”双层方案，并对窗口外历史对话进行摘要压缩，兼顾 token 成本与上下文完整性。
- 引入 Redis 问答缓存与异步处理机制，提升高频问题响应速度并降低重复推理成本。

## 项目亮点
- 将大模型调用从“单点 API”升级为“统一治理层”，支持多 Key、权重调度、基于并发负载的路由选择与熔断保护。
- 构建可观测的智能问答系统，支持健康检查、路由状态监控、QA 分析看板与用户反馈闭环。
- 面向海关业务的高风险问答场景，强化证据引用和保守拒答策略，提升系统可靠性与可解释性。
